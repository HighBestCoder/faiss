## 待探索优化方向

### 1. 更激进的预取策略

**思路:** 当前只预取当前节点的邻居向量，可以更激进地预取"邻居的邻居"。

```cpp
// 当前: 预取 neighbors[i] 的向量
// 改进: 同时预取 neighbors[i] 的邻居列表
for (int i = 0; i < num_neighbors; i++) {
    // 预取邻居向量 (已有)
    _mm_prefetch(vector_data + neighbors[i] * dim, _MM_HINT_T0);
    // 新增: 预取邻居的邻居列表
    _mm_prefetch(neighbor_list_ptr(neighbors[i]), _MM_HINT_T1);
}
```

**预期收益:** 5-10% (减少邻居列表访问的 cache miss)

**风险:** 可能污染 cache，需要调优预取距离

---

### 2. SIMD 优先队列

**思路:** HNSW 搜索的热点是维护 top-k 候选集的堆操作。用 AVX-512 并行比较可加速。

```cpp
// 当前: std::priority_queue，逐个比较
// 改进: 用 SIMD 同时比较 16 个候选
__m512 candidates = _mm512_loadu_ps(candidate_distances);
__m512 threshold = _mm512_set1_ps(heap_top);
__mmask16 mask = _mm512_cmp_ps_mask(candidates, threshold, _CMP_LT_OS);
if (mask) {
    // 有更好的候选，更新堆
}
```

**预期收益:** 10-15% (heap 操作占搜索时间 ~20%)

**难度:** 高，需要重写堆数据结构

---

### 3. 图压缩

**思路:** 减少图结构的内存占用，提高 cache 效率。

选项:
- **邻居 ID 压缩:** 用 delta encoding + varint，节省 50%+ 空间
- **移除冗余边:** 剪枝低效边，减少遍历
- **分层存储:** 热点节点放 L1，冷节点放 L3

```cpp
// 当前: 每个邻居 ID 用 4 字节 (int32)
// 改进: delta encoding，平均 1-2 字节
// neighbors: [1000, 1005, 1003, 1050]
// delta:     [1000,    5,   -2,   47]  -> varint 编码
```

**预期收益:** 5-10% (减少内存带宽)

**风险:** 解码开销可能抵消收益

---

### 4. 搜索路径缓存

**思路:** 热点入口节点的搜索路径可以缓存复用。

```cpp
// 缓存从 entry_point 到各层的常用路径
struct PathCache {
    std::vector<storage_idx_t> cached_path[MAX_LEVEL];
    float cached_query[DIM];  // 上次查询向量
    float similarity_threshold = 0.9;  // 相似度阈值
};

// 如果新查询与缓存查询相似，复用路径
if (dot_product(query, cached_query) > similarity_threshold) {
    start_from_cached_path();
}
```

**预期收益:** 对相似查询可达 20-30%

**适用场景:** 查询有局部性时 (如推荐系统)

---

### 5. Level 0 专用优化

**思路:** HNSW 搜索 90%+ 时间在 level 0，针对性优化。

优化点:
- **Level 0 专用内存布局:** 连续存储，减少跳转
- **Level 0 批量距离计算:** 一次计算所有邻居距离
- **Level 0 SIMD 遍历:** 用 AVX-512 并行处理 16 个邻居

```cpp
// Level 0 专用搜索函数
void search_level0_optimized(const float* query, ...) {
    // 所有邻居向量连续存储
    // 用 batch_16 一次计算所有距离
    float distances[16];
    fvec_L2sqr_batch_16(query, neighbor_vectors, dim, distances);
    
    // SIMD 找最小值
    __m512 d = _mm512_loadu_ps(distances);
    int min_idx = _mm512_reduce_min_ps_idx(d);
}
```

**预期收益:** 15-25% (level 0 是热点)

**难度:** 中等，需要修改内存布局

---

### 6. 查询批处理优化

**思路:** 多个查询共享相同的图遍历路径前缀。

```cpp
// 当前: 每个查询独立搜索
// 改进: 批量查询共享上层遍历
void batch_search(const float* queries, int nq, ...) {
    // Step 1: 所有查询共享从 entry_point 到 level 1 的路径
    auto common_entry = find_common_entry(queries, nq);
    
    // Step 2: 在 level 0 并行搜索
    #pragma omp parallel for
    for (int q = 0; q < nq; q++) {
        search_level0(queries + q * dim, common_entry, ...);
    }
}
```

**预期收益:** 10-20% (减少重复遍历)
